# 目的

ユーザーが「今晩の夕飯を提案して」と依頼したとき、**気分や体調**を起点にヒアリングし、
5人家族を想定した夕食レシピを 2〜3 件提案する。
選ばれたレシピについて、5人分の材料リストと簡易調理手順を出力する。

---

# ヒアリングフロー

以下の質問を **1 回のインタラクション**（`ask_question` ツール）でまとめて聞く。
回答がない項目はデフォルト値を適用する。

## Q1. 今日の気分・体調（必須）

> 今の気分や体調に一番近いものを選んでください。

| 選択肢 | 内部タグ | 提案方針 |
|--------|----------|----------|
| 疲れた・ヘトヘト | `tired` | 時短（30分以内）、消化にやさしい、ビタミン B 群多め |
| さっぱりしたい | `light` | 酢の物・サラダ系、低脂質、冷製メニュー可 |
| がっつり食べたい | `hearty` | ボリューム重視、肉・揚げ物 OK、炭水化物多め |
| 辛いものが食べたい | `spicy` | 麻婆豆腐・カレー系、唐辛子・スパイス |
| ほっこり温まりたい | `comfort` | 煮込み・鍋・シチュー系 |
| 特になし | `any` | 季節のおすすめ |

## Q2. ジャンル（任意）

> 和食・洋食・中華・エスニック・なんでも OK

デフォルト: `なんでも OK`

## Q3. アレルギー・NG 食材（任意）

> アレルギーや苦手な食材があれば教えてください。

デフォルト: なし

## Q4. 調理時間の上限（任意）

> 調理にかけられる時間は？（例: 30分、60分）

デフォルト:
- `tired` → 30分
- その他 → 60分

## Q5. 冷蔵庫にある食材（任意）

> 使い切りたい食材があれば教えてください。

デフォルト: 指定なし（AI が季節食材で提案）

# レシピ提案ルール

1. **人数は 5 人を前提**にする。材料の分量はすべて 5 人分で記載する。
2. 気分タグ × ジャンル × 季節（現在月）でレシピを絞り込む。
3. NG 食材が `ingredients` に含まれるレシピは除外する。
4. 冷蔵庫食材が指定されたら、その食材を活用するレシピを優先する。
5. 候補は **2〜3 件**を表形式で提示する。

## 提示フォーマット

```markdown
### 🍽️ 今晩のおすすめ（5人分）

| # | レシピ名 | ジャンル | 所要時間 | ポイント |
|---|----------|----------|----------|----------|
| 1 | ○○ | 和食 | 25分 | 疲労回復にビタミン B₁ 豊富 |
| 2 | △△ | 洋食 | 35分 | 野菜たっぷりで胃にやさしい |
| 3 | □□ | 中華 | 20分 | 冷蔵庫の鶏肉を活用 |

> 番号を選んでください。詳しいレシピを表示します。
```

# レシピ詳細の出力フォーマット

ユーザーが番号を選んだら、以下の形式で出力する。

```markdown
## 【レシピ名】（5人分・所要時間 ○分）

### 材料

| 材料 | 分量（5人分） |
|------|---------------|
| 鶏もも肉 | 500g |
| 玉ねぎ | 2個 |
| … | … |

### 作り方

1. **下ごしらえ** – 鶏もも肉は一口大に切り、塩こしょうをふる。
2. **炒める** – フライパンに油を熱し、玉ねぎを中火で炒める。
3. …
4. …

### 💡 ワンポイント

- ○○を加えると風味がアップします。
- 残った△△は翌日のお弁当にも使えます。

### 🥗 おすすめ副菜

- □□のサラダ（5分で完成）
- ◇◇のスープ
```

# 季節食材ガイド（参考）

レシピ提案時に季節の旬食材を優先的に取り入れる。

| 月 | 旬の食材（例） |
|----|----------------|
| 1-2月 | 大根、白菜、ブリ、牡蠣 |
| 3-4月 | 新玉ねぎ、春キャベツ、タケノコ、鯛 |
| 5-6月 | アスパラガス、トマト、カツオ、そら豆 |
| 7-8月 | なす、きゅうり、枝豆、鰻 |
| 9-10月 | さつまいも、きのこ類、サンマ、栗 |
| 11-12月 | 白菜、れんこん、鮭、みかん |

# 備考

- 人数の既定値は **5人** だが、ユーザーが変更を希望した場合は柔軟に対応する。
- 本スキルは `.agent/skills/dinner-suggestions/` に配置する。
- レシピデータを拡充する場合は `resources/recipes.json` に JSON で分離し、SKILL.md から参照する。
